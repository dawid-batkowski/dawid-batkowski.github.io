P5 - Spite: The Depth of Darkness

Spite: TDOD - Project Pitch

"As a blacksmith, you return to your home town and are shocked to find it recently abandoned and overrun by trolls. Bash your way through the hoardes with brawn and skill in search of answers which all lead to the church and their secrets."

Spite: TDOD - Overview

Specifications:
- Engine: TGE (Custom DX11)
- Platform: PC
- My Role: Technical Art, Animation, VFX

This was the 5th group project developed in TGE. Inspired themed after Diablo 3, the project focuses on exploration and combat. The game features four distinct levels, culminating in a boss fight. Player character is equipped with left mouse attack and 5 unique abilities, unlockable throughout the gameplay. Additionally, game features 2 enemy types, elite enemy variant with random modyfiers and boss.

My role was heavily focused around rigging and animating with my main responsibility being player character. My second role was to create enviroment/UI shaders. These included fogs, resource orbs, vignette, color grading, shader for tilleable textures. Lastly, I was also responsible for a lot of VFX. These included effects for some player abilities, pick ups and enviroment.



Featured Work - Health & Rage resource orbs

A very important shader I was responsible for were resource orbs. Since this shader would be visible at all time throughout the gameplay it took more time to complete than any other effect made during this project. The goal was to create two resource orbs in similar style but distinct look to represent health and rage meter. My restrictions for this shader were: plane as geometry for each orb, single texture slot, no blending modes other than opaque with alpha clipping.

Creation of this shader started with preparing main shape, using some simple math based on modyfing length of UVs and discarding pixel below certain treshold I was quickly able to get main shape for each orb. Next step was to create some flow patterns. For rage orb I decided to go with something re-assmbling flames with it's color and chaotic movement. For health orb I decided to go with more tame, liquidy, with slower flow, red in color and a faint pulsing light in the mdidle to sigdnify beating heart. The main pattern for both orbs was created using multiple noise textures created in IlluGen and using one noise texture to modulate UVs of another noise texture. Additional UV modyfications were used to make a flow more spherical to give the orb it's round perspective. Coloring in both cases was done using 3-way lerp and hand-picked values.

After pattern and coloring was done, it was time to fake some ligthing. The orbs feature fake directional light, rim light and specular light to further enchance the 3D look of it and fit better in a lit game enviroment.

Last visual part was to create a background for when orb is not fompletely full. For that I have created a scratch mark mask in Illugen, which was included in a noise texture. In a shader a color was assigned ot that mask and the visiblity of a background was controlled via smoothstep Y axis of moving UVs controlled by a contant buffer variables.

Health & Rage resource orbs - early iterations

Creating this shader was a very iterative process.

Rage orb's initial design assumed a creepy face to appear when orb was full. Design was later changed due to feedback mentioning that it look very halloween-like.

Second design leaned more into flame direction, using multiple noise texture masks it was possible to give the orb more varied and chaotic movement with more dynamic colors. It wasn't untill moving noise masks were used to modulate other moving noises that I finally was happy with design. From that point a center piece of the orb was added in form of a shining light and some small tweaks were made regarding types of noise used, ways of layering noises, and balancing colors.

Health orb is using rage orb's shader at it's heart. The shader was however modified to create more calm, flowing, liquidy and deeper effect with stronger panning towards the side of the orb rahter than going up.

The initial final result assumed usage of 3 textures. One for noise masks, background and gradient atlas for coloring. Due to implementation limitations the shader had to be modyfied to use only one texutre. This texture included 3 masks for nosies and 1 mask for background pattern.

Featured Work 2 - Health pickup

My second major work during this project was creation of pickup health orb . One of the bigger limitations when creating VFX during this project was lack of particle system.

And while I was designing a concept for that effect I was thinking, what would I want from particle system to be able to do for this effect to be finished?
It spawns quads over time, the quads move in either one or multiple directions, there is a lifetime variable I can use to control pixel shader, there is no visible sign of particles poping in and out of existance without any smooth transition, they billboard towards camera.

All of these attributes can be done using only vertex and pixel shader. All of them except for spawning geometry, however, as long as effect is continous and it isimpossible to see whether a geometry is actually spawned or just moved back and forth, a very convincing fake can be made.

Additional Work - Flipbook VFX

Health-Based Damage Visualization was implemented by using multiple noise functions with a health parameter. As health decreases, characters appear progressively more damaged and blood-covered.

Water Shader was implemented using depth comparisons to simulate foam and water depth. A noise texture created in Substance Designer was used to simulate wave motion. Additionally, shadow map distortion was applied to enhance the illusion of moving water.

Cloud Shader was a lightweight but effective addition. A large plane was placed above the level and animated using panning noise with aggressive alpha clipping. Combined with existing shadow casting and the fixed camera angle (which never looks upward), this created a convincing illusion of moving clouds casting shadows.

Hit Effect for both enemies and the player was implemented using a Fresnel-based shader, combined with time-controlled parameters via constant buffers to produce a reactive flash effect.

Fire Shader for campfires and braziers combines vertex displacement, panning noise for color variation, and height-based alpha clipping. Alpha clipping was chosen over traditional transparency to avoid sorting issues and better match the stylized aesthetic.

Post-Processing Effects were implemented in a fullscreen pass to enhance the overall visual presentation. These included vignette and color grading adjustments.

Fog Planes were placed throughout the level to enhance depth and atmosphere. The shader uses height-based calculations between the fog plane and scene vertex position, combined with panning noise to create a more dynamic and less uniform effect.
